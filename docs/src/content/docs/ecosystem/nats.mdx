<Aside type="note">
NATS is optional. Nerva works without it for single-node deployments. Add NATS when you need multi-node coordination.
</Aside>

NATS JetStream provides the transport layer for Nerva when running as a distributed system. It handles message routing between nodes, persistent key-value storage, and pub/sub event delivery.

## What NATS provides

| Capability | NATS feature | Nerva usage |
|-----------|-------------|-------------|
| Pub/sub messaging | Core NATS subjects | Event distribution between nodes |
| Persistent streams | JetStream | Reliable delivery with replay |
| Key-value store | JetStream KV | Shared state (nerve status, config, memory) |
| Request/reply | Core NATS | Synchronous inter-node queries |

## Configuration

Set the NATS connection URL in your environment or configuration:

```bash
export NATS_URL="nats://localhost:4222"
```

For authenticated clusters:

```bash
export NATS_URL="nats://user:password@nats.example.com:4222"
```

## Transport abstraction

Nerva uses a transport abstraction layer so the same code works with NATS in production and an in-memory `FakeTransport` in tests. All transport access goes through three operations:

```python
# Publish an event to a subject
await transport.publish("events.nerve.completed", payload)

# Read from key-value store
value = await transport.kv_get("nerve:weather-lookup:status")

# Write to key-value store
await transport.kv_set("nerve:weather-lookup:status", "active")
```

Never use raw NATS client calls directly — always go through the transport interface.

## Key-value namespaces

| Prefix | Contents |
|--------|----------|
| `nerve:<name>:*` | Nerve metadata, status, calibration |
| `memory:hot:*` | Hot memory (session state) |
| `memory:warm:*` | Warm memory (cross-session facts) |
| `config:*` | Runtime configuration |

## Pub/sub subjects

| Subject pattern | Description |
|----------------|-------------|
| `events.nerve.*` | Nerve lifecycle events (registered, invoked, completed) |
| `events.brain.*` | Brain orchestration events (think, dispatch, plan) |
| `events.system.*` | System events (boot, shutdown, health) |

## Running NATS locally

```bash
# Start NATS with JetStream enabled
nats-server --jetstream

# Or via Docker
docker run -p 4222:4222 nats:latest --jetstream
```

## Testing without NATS

In tests, use `FakeTransport` which provides the same interface backed by in-memory dictionaries. No NATS server required:

```python
from nerva.transport import FakeTransport

transport = FakeTransport()
await transport.kv_set("key", "value")
result = await transport.kv_get("key")
assert result == "value"
```

<Aside type="tip">
FakeTransport is all you need for unit tests. Reserve NATS integration tests for CI or staging.
</Aside>
