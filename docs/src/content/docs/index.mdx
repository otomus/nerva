## Why Nerva?

SDKs are too thin — you rebuild routing, execution, tools, and memory every time.
Frameworks own your architecture — swap a component and you fight the framework.

Nerva is the middle ground. Opinionated primitives, zero opinions on how you compose them.

<CardGrid>
  <Card title="ExecContext" icon="puzzle">
    Request identity, permissions, memory scope, and tracing — threaded through every primitive.
  </Card>
  <Card title="Router" icon="magnifier">

  </Card>

## How It Works

```
         Message
           |
    +------v------+
    | ExecContext  | <-- user identity, permissions, memory scope, trace ID
    +------+------+
           |
    +------v------+
    |   Policy     | <-- rate limit, budget check, approval gate
    +------+------+
           |
    +------v------+
    |    Router    | <-- classify intent, select handler (from Registry)
    +------+------+
           |
    +------v------+
    |   Runtime    | <-- execute handler (subprocess / in-process / container)
    |     +- Tools | <-- discover from Registry, sandbox, call
    |     +- Memory| <-- read context, write results
    +------+------+
           |
    +------v------+
    |  Responder   | <-- format for channel + tone
    +------+------+
           |
         Output
```

## Library, Not Server

Nerva runs **inside** your web framework. It does not own your HTTP layer, your auth, or your API docs.

```
FastAPI / NestJS / Express   (HTTP, auth, sessions, swagger)
  +-- Nerva                  (routing, runtime, tools, memory, policy)
       +-- LLM providers, MCP servers, subprocess agents
```
