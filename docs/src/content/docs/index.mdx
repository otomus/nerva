Every agent framework wants to own your stack. Nerva doesn't. It gives you **8 composable primitives** — routing, runtime, tools, memory, policy, and more — that drop into the web framework you already use. No lock-in, no opinions about your HTTP layer, no hidden abstractions. Build your agent system, not the plumbing around it.

<Aside type="tip" title="Library, not framework">
Nerva runs **inside** your web framework. It does not own your HTTP layer, your auth, or your API docs. You bring FastAPI, NestJS, or Express — Nerva handles the agent infrastructure underneath.
</Aside>

## The 8 Primitives

Everything you need, nothing you don't. Each primitive is independent, composable, and replaceable.

<CardGrid stagger>
  <LinkCard title="ExecContext" description="Request identity, permissions, memory scope, and tracing — threaded through every primitive." href="/nerva/docs/primitives/context/" />
  <LinkCard title="Router" description="Classify intent and select the right handler. Embedding, LLM, rule-based, or hybrid strategies." href="/nerva/docs/primitives/router/" />
  <LinkCard title="Runtime" description="Execute agent code with isolation, timeouts, circuit breakers, and streaming support." href="/nerva/docs/primitives/runtime/" />
  <LinkCard title="Tools" description="Discover, sandbox, and invoke external tools. MCP servers, functions, composites." href="/nerva/docs/primitives/tools/" />
  <LinkCard title="Memory" description="Tiered storage — hot (session), warm (episodes and facts), cold (vector search)." href="/nerva/docs/primitives/memory/" />
  <LinkCard title="Responder" description="Format agent output for the target channel and tone." href="/nerva/docs/primitives/responder/" />
  <LinkCard title="Registry" description="Unified catalog of agents, tools, and plugins. Register, discover, health-check." href="/nerva/docs/primitives/registry/" />
  <LinkCard title="Policy" description="Declarative safety, permissions, rate limits, cost budgets, and approval gates." href="/nerva/docs/primitives/policy/" />
</CardGrid>

<Aside type="note" title="Use what you need">
You don't need all 8. Use one primitive, use all of them, or replace any with your own implementation. Every primitive has a clean interface you can swap.
</Aside>

## How It Works

<object type="image/svg+xml" data="/nerva/docs/architecture-flow.svg" style="width:100%;max-width:800px;margin:2rem auto;display:block;">Architecture flow diagram</object>

```
FastAPI / NestJS / Express   (HTTP, auth, sessions, swagger)
  └── Nerva                  (routing, runtime, tools, memory, policy)
       └── LLM providers, MCP servers, subprocess agents
```

## What's in the Box

Each primitive ships with multiple built-in strategies. Pick the one that fits, or write your own.

<CardGrid>
  <Card title="4 Router Strategies" icon="magnifier">
    Embedding similarity, LLM classification, rule-based pattern matching, and hybrid (combine any).
  </Card>
  <Card title="3 Runtime Modes" icon="rocket">
    In-process, subprocess isolation, and container sandboxing — with circuit breakers on all three.
  </Card>
  <Card title="3 Tool Managers" icon="setting">
    MCP server pools, plain function registries, and composite tool chains.
  </Card>
  <Card title="3 Memory Tiers" icon="document">
    Hot (session state), warm (episodes and extracted facts), cold (vector search over history).
  </Card>
  <Card title="3 Responder Types" icon="pencil">
    Plain text, structured JSON, and streaming — each with channel-aware formatting.
  </Card>
  <Card title="3 Registry Backends" icon="list-format">
    In-memory, file-based, and remote (HTTP/gRPC) — all behind the same interface.
  </Card>
  <Card title="3 Policy Engines" icon="approve-check">
    Static rules, OPA integration, and LLM-as-judge — stack them for defense in depth.
  </Card>
</CardGrid>

<Aside type="tip" title="Scaffold in seconds">
```bash
npx nerva new my-agent --lang python
```
The CLI generates a working project with your chosen primitives, a test suite, and a dev server. Supports `--lang python`, `--lang typescript`, and `--template` for common patterns.
</Aside>
